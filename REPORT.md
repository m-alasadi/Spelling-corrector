## 4. محرك التصحيح الإملائي — Spell Checker Engine

### 4 تحسينات مؤسسية (Enterprise Optimizations)

| # | التحسين | الوصف | الأثر |
|---|---------|-------|-------|
| 1 | **التصفية المسبقة** | تخطي الضوضاء/الرموز/الإيموجي قبل إرسالها للـ AI | توفير 30-50% من تكاليف API |
| 2 | **الذاكرة الدلالية** | ذاكرة مؤقتة SQLite بمستويين (ذاكرة + قرص) | تكلفة صفر للنصوص المتكررة |
| 3 | **تحديد المعدل** | Semaphore(5) يحد من الطلبات المتزامنة | منع HTTP 429 |
| 4 | **المعالجة الذكية** | معالجة فقط للمقاطع التي تحتاج AI، دفعات بحجم 30 | سرعة ×5 |

### مسار التصحيح بالتفصيل

```
النص المدخل
    │
    ▼
┌─────────────────────────┐
│  1. Pre-Filter (تصفية)   │  ← تخطي الضوضاء، الأرقام، الإيموجي
│     should_skip_ai()     │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│  2. Cache Check (ذاكرة)  │  ← L1: ذاكرة RAM (فوري)
│     SemanticCache.get()  │  ← L2: SQLite (سريع)
└──────────┬──────────────┘
           │ (لم يُعثر عليه)
           ▼
┌─────────────────────────┐
│  3. Dict Check (قاموس)   │  ← مطابقة دقيقة من custom_dict.txt
│     _correct_with_dict() │  ← 15 قاعدة + 106 كلمة معروفة
└──────────┬──────────────┘
           │ (لم يُعثر عليه)
           ▼
┌─────────────────────────┐
│  4. AI Check (ذكاء)      │  ← GPT-4o-mini
│     _call_ai_batch()     │  ← دفعات بحجم 30 مع Semaphore(5)
│     temperature: 0.1     │  ← 3 محاولات مع exponential backoff
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│  5. Word Diff (مقارنة)   │  ← difflib.SequenceMatcher
│     compute_word_diff()  │  ← مقارنة كلمة بكلمة
└──────────┬──────────────┘
           │
           ▼
      النتيجة النهائية
```

### إعدادات الذكاء الاصطناعي (AI Configuration)

| الإعداد | القيمة | السبب |
|---------|--------|-------|
| **النموذج** | GPT-4o-mini | التكلفة/الأداء المثالي |
| **Temperature** | 0.1 (إملائي) / 0 (نحوي) | قابلية التكرار |
| **Max Tokens** | 2048 (إملائي) / 4000 (نحوي) | كفاية للنصوص الطويلة |
| **Prompt Strategy** | دفعات مرقمة `[1] [2]...` | معالجة متعددة في طلب واحد |
| **Retry** | 3 محاولات + exponential backoff | موثوقية عالية |

## 5. التدقيق النحوي — Grammar Check (ميزة جديدة)

### التصميم

| العنصر | التفاصيل |
|--------|----------|
| **النوع** | On-Demand (عند الطلب فقط) |
| **التفعيل** | تلقائي عند فتح تبويب "📐 نحوي ✨" |
| **الحماية** | لا يُعدّل على الكلمات العامية مطلقاً |
| **العرض** | خطوط زرقاء مزدوجة (double underline) |

### قواعد حماية اللهجة العامية (مُحدّثة)

```
⚠️ أهم قاعدة:
أي كلمة عامية أو لهجية يُمنع منعاً باتاً تعديلها أو استبدالها.

أمثلة على كلمات يُمنع تعديلها:
├─ عندنا ← لا تحوّلها إلى "لدينا" أو "أن"
├─ هسه   ← لا تحوّلها إلى "الآن"
├─ شنو   ← لا تحوّلها إلى "ماذا"
├─ وين   ← لا تحوّلها إلى "أين"
├─ هاي   ← لا تحوّلها إلى "هذه"
├─ شلون  ← لا تحوّلها إلى "كيف"
└─ اريد  ← لا تحوّلها إلى "أريد"
```

### ما يُصحح نحوياً (فقط)

| الخطأ | التصحيح | مثال |
|-------|---------|------|
| فاعل مرفوع | إضافة ضمة | الرجلُ (لا الرجل) |
| مفعول به منصوب | إضافة فتحة | البيتَ (لا البيت) |
| ضمائر الرفع/النصب | تصحيح الضمير | رأيتُهُ (لا رأيته) |
| الإضافة | تصحيح المضاف إليه | كتابُ الطالبِ (لا كتاب الطالب) |

### مثال على النتيجة

```json
{
  "original": "جاء الرجل البيت ورایت الرجل عندنا",
  "corrected": "جاء الرجلُ البيتَ ورأى الرجلُ عندنا"
}
```

ملاحظة: "عندنا" **لم تتغير** ✅

## 8. قواعد البيانات (Databases)

### 1. ذاكرة التصحيح المؤقتة (cache.db)

```sql
corrections (
  text_hash TEXT PRIMARY KEY,  -- MD5 للنص
  original TEXT NOT NULL,       -- النص الأصلي
  corrected TEXT NOT NULL,      -- النص المصحح
  model TEXT DEFAULT '',        -- النموذج المستخدم
  created_at TIMESTAMP
)
```

### 2. قاعدة الأخطاء (error_db)

```sql
corrections (
  id INTEGER PRIMARY KEY,
  original TEXT,                -- الكلمة الأصلية
  corrected TEXT,               -- الكلمة المصححة
  context TEXT,                 -- سياق الجملة
  source TEXT,                  -- 'ai' أو 'user'
  frequency INTEGER DEFAULT 1,  -- عدد مرات الخطأ
  created_at TIMESTAMP,
  UNIQUE(original, corrected)
)
```

### 3. القاموس المخصص (custom_dict.txt)

```
# قواعد: خطأ => صحيح
المشروووع => المشروع
ابو => أبو

# كلمات معروفة: word <= word
الحيات <= الحياة
```
